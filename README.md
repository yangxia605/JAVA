#### JDK相关的基本概念
###### JDK:是Java Development Kit, 即Java开发工具包，用于Java开发程序
###### JRE: Java Runtime Environment, 即 Java运行环境，用于运行Java程序
###### JVM：Java Virtual Machine, Java虚拟机。将Java程序编译成机器语言，实现Java层序的跨平台性
###### Note：JVM是用来保证Java程序跨平台的，它本身是不能跨平台的
###### 开发工具JDK
###### JDK的下载官方地址：https://www.oracle.com/downloads/
__________________________________________________________________________________


###### Java程序的最小单位是类，一个Java程序至少拥有一个类
###### .idea目录和.iml文件：IDEA开发工具使用的配置文件，我们不需要操作，可以隐藏
###### src，source单词的缩写，即我们编写代码源文件放在这个目录下
###### out目录：Java程序的输出目录，Java程序运行的时候，生成后缀名为.class 的文件存放在该目录下
###### 点击运行按钮，生成.class文件叫做字节码文件，这个过程叫编译
###### External Libraries：扩展类库，即Java程序编写和运行所依赖的JDK中的类
###### Java程序开发与运行原理：.java源文件在java.exe的编译下，生成.class字节码文件，再通过java.exe运行出结果

#### 命名规范：
###### （1）类和接口：首字母大写，如果有多个单词，每个单词首字母大写：HelloWorld、Student；
###### （2）变量和方法：首字母小写，如果有多个单词，从第二个单词开始首字母大写：getName
###### （３）常量名：所有字母都大写，多个单词用下划线隔开：ＭＡＸ＿ＶＡＬＵＥ
###### （４）包名：全部小写，如果有多级，用点号隔开：ｃｎ．ｄｅｍｏ
###### 　包名其实就是文件夹，用来区分重名类的

#### 方法重载
###### 什么是方法重载？在同一个类中的多个方法，他们的方法名相同，参数列表不同，并且与返回值的类型无关
######  为什么需要方法重载？当实现的功能相同，但具体的实现方式不同时，我们可以通过定义名称相同，参数（条件）不同的方法，来更好的识别和管理类中的方法
###### (1)参数个数不同 (2)参数类型不同 （不用同时满足）

#### 数组
###### 为什么需要数组？为了存储多个数据值
###### 什么是数组？数组是用来存储同一种数据类型多个元素的容器
###### 数据类型：可以是基本类型，也可以是引用类型
###### 数组的定义格式：（1）数据类型[] 数组名 = new 数据类型[长度]，数组长度才定义时，不可更改。比如：int [] arr = new int[3] //定义一个长度为3的整型数组
###### （2）数据类型[] 数组名 = new 数据类型[] {元素1，元素2，元素3...}，好处是：定义时元素是确定的，避免内存空间的浪费.比如：int[] arr2 = new int[]{1,2,3}
###### (3)数据类型[] 数组名 = {元素1，元素2，元素3...} 比如：int[] arr3 = {1,2,3}
###### 数组的访问方式：通过数组的索引访问数组的元素,索引从0开始，往后依次类推
###### 数组中未手动赋值的元素有默认值：int是0
###### 数组的遍历
###### 数组的初始化：在内存中为数组开辟连续空间并为每个元素赋值的过程
###### 动态初始化：只给定指定长度，由系统给出默认值，其中默认值，整数型：0；浮点型：0.0；字符型：空字符；布尔型：false；引用类型：null.比如：int[] arr = new int[5]
###### 静态初始化： 给出初始化值，由系统决定数组长度.比如：int[] arr = new int[]{1,2,3}或者 int[] arr = {1, 2, 3}
###### 数组索引越界异常：ArrayIndexOutOfBoundsException（当访问了不存在的索引时）
###### 空指针异常：NullPointerException（数组引用存储的值为null而非数组的地址值）比如：int[] arr1 = new int[2];arr1[0]=1;arr1[1]=2;arr1 = null;此时想要打印 arr1[1]就会报错
###### 引用类型传递和基本类型传递：引用类型的的变量作为参数传递给方法时，传递的是地址值;基本类型传递的是值

#### JAVA面向对象
###### 特点(1)是一种更符合人们思考习惯的思想；（2）把复杂的事情简单化；（3）把人们从执行者变成了指挥者
###### 类与对象：（1）类主要由属性和行为构成，是一系列具有相同属性和行为的事物的统称；（2）:抽象是把一系列相关事物共同的属性和行为提取出来的过程
###### （3）某一类事物的具体存在；（4）类是一个抽象的概念，对象是该类事物的具体体现，是一种具体的存在
###### 如何定义一个类？就是把一系列相关事物的属性和行为抽取出来的过程：事物的属性：在类中叫成员变量；事物的行为：在类中叫成员方法
###### 如何创建一个对象？类名 对象名 = new  类名（）;
###### java中使用变量的规则：就近原则
###### 局部变量（方法的参数列表或者方法内部）和成员变量（类中或者方法外）
###### 成员变量有默认初始化值，局部变量没有默认初始化值，必须先赋值再使用
###### 成员变量的作用范围在类中；局部变量的作用范围在方法中
###### 成员变量在堆内存，局部变量在栈内存
######  成员变量随着对象的创建而存在，随着对象的消失而消失；局部变量随着方法的调用而存在；随着方法调用完毕而消失
###### 局部变量和成员变量重名时，采用就近原则
